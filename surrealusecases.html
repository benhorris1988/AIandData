Staging & Transformation Workloads

Pipeline Modernization
Priority: High
Difficulty: Medium
01

Move Staging & Apply Constraints

Move massive staging payloads into SurrealDB. Test its ability to seamlessly ingest and map a diverse range of existing database types and structures. Apply strict schema definitions, complex indexes, and heavy backend database constraints to validate structural integrity.

02

Heavy Transformation & Dimension Modeling

Select complex ETL jobs that currently take hours to process in SSIS/SQL. Rebuild them natively in SurrealDB to test execution performance. Critically, evaluate if SurrealQL can elegantly handle complex enterprise modeling paradigms like Temporal Tables and Slowly Changing Dimensions (SCDs).

Target Architectural Outcome

Prove that SurrealDB can rapidly ingest and heavily transform vast datasets faster than our legacy pipelines, and mathematically validate if that resulting transformed data is highly performant to direct analytical queries.

Graph-Based Data Provisioning

Security & Access Control
Priority: High
Difficulty: High
01

Acknowledge the SQL Constraint

Currently, implementing dynamic row, column, and schema security in SQL Server requires complex mapping tables, heavy triggers, and nested views that are difficult to audit and degrade performance at scale.

02

Model Security as a Graph

Instead of mapping tables, natively construct physical edge relationships connecting a `User` node to a `Dataset` node via a directed edge (e.g., `HAS_ACCESS_TO`).

03

Define Permissions

Utilize SurrealDB's native DEFINE PERMISSIONS functionality to execute this graph logic directly at the database layer securely.

-- RLS enforced natively by Graph Edges in SurrealQL
DEFINE TABLE dataset SCHEMAFULL
  PERMISSIONS
    FOR select WHERE $auth.id ->has_access_to->id;

Target Architectural Outcome

Decouple massive amounts of access control logic from the API and provisioning tiers by securely enforcing dynamic data-mesh security natively within the SurrealDB intelligence layer.

AI & Semantic Search
Priority: High

01

Baseline the Relational Data

Extract a frequently queried, highly-structured subset of historical data from our existing enterprise GOLD lakehouse layer.

02

Vectorization & Graph Linking

Convert the flat dimension tables into a unified Graph structure, and simultaneously generate numerical Vector embeddings for all unstructured free-text columns (like logs, histories, or notes).

Graph Performance Benchmarking

Scale & Viability
Priority: Medium
Difficulty: Medium
01

Establish the Test Scenario

Construct a complex, deep recursive entity traversal pattern that pushes traditional SQL to its limit (e.g., massive supply chain traceability or fraud detection networks).

02

Parallel Execution Protocol

Build the exact identical test framework using native Microsoft Fabric Graph extensions, Kusto Query Language (KQL), and SurrealQL, ensuring parity in data volume.

Target Architectural Outcome

Extract objective baseline metrics for maximum Ingestion Throughput, raw compute efficiency, developer syntax maintainability, and total multi-hop Query Latency at scale.

Schema-less Telemetry Logging

Document Store Capabilities
💡 Stu's Idea
Priority: Low
Difficulty: Low
01

Target Volatile Ingestion Sources

Identify extremely rapid and unstructured data streams suffering from schema rot, such as IoT sensor telemetry, rapid application logs, or unpredictable 3rd-party JSON feeds.

02

Leverage the Document Engine

Directly funnel these JSON payloads into SurrealDB document-store tables explicitly without defining rigid structural schemas, forcing the database to dynamically adapt.

Target Architectural Outcome

Benchmark SurrealDB's dynamic auto-indexing capabilities against SQL Server's rigid OPENJSON syntax when probing nested unstructured attributes, testing for upstream pipeline resilience.

Real-time "Live" Dashboards

Event Streaming
Priority: Low
Difficulty: Medium
01

Identify Transport Overhead

Isolate an operational 'Command Center' style dashboard currently relying heavily on taxing database polling routines or extensive SignalR/event-bus architectures for live refreshes.

02

Establish Persistent WebSockets

Deploy a frontend client utilizing a SurrealDB LIVE SELECT connection. As background systems ingest/alter operational database records, verify immediate push notifications propagating to the frontend browser.

Target Architectural Outcome

Prove that we can substantially streamline infrastructure by allowing the operational database cluster itself to natively act as a resilient pub-sub message broker for UI components.
